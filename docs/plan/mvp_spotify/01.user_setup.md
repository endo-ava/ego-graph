# EgoGraph セットアップガイド

## 概要
このガイドでは、EgoGraph Spotify MVPを動作させるために必要な初期セットアップ手順を説明します。

## 前提条件
- GitHubアカウント
- Spotifyアカウント（Premium不要、無料アカウントでOK）
- 基本的なコマンドライン操作の知識

## セットアップの流れ

### 推定所要時間: 1-2時間

1. **APIクレデンシャルの取得**（45-60分）
   - Spotify Developer Dashboardでアプリ作成
   - Nomic Atlasでアカウント作成とAPIキー取得
   - Qdrant Cloudでクラスタ作成

2. **GitHub Secretsの設定**（15-20分）
   - リポジトリにシークレットを追加

3. **GitHub Actionsの有効化とテスト**（10-15分）
   - ワークフローの手動実行
   - ログの確認

4. **動作確認**（5-10分）
   - Qdrantダッシュボードでデータを確認

## セットアップチェックリスト

コピーして使用してください：

```
□ Spotify Developer Dashboardでアプリを作成した
□ Spotify Client IDを取得した
□ Spotify Client Secretを取得した
□ Spotify Refresh Tokenを取得した（OAuth認証完了）
□ Nomic Atlasでアカウントを作成した
□ Nomic API Keyを取得した
□ Qdrant Cloudでアカウントを作成した
□ Qdrant Cloudでクラスタを作成した
□ Qdrant URLを取得した
□ Qdrant API Keyを取得した
□ GitHub SecretsにSPOTIFY_CLIENT_IDを追加した
□ GitHub SecretsにSPOTIFY_CLIENT_SECRETを追加した
□ GitHub SecretsにSPOTIFY_REFRESH_TOKENを追加した
□ GitHub SecretsにNOMIC_API_KEYを追加した
□ GitHub SecretsにQDRANT_URLを追加した
□ GitHub SecretsにQDRANT_API_KEYを追加した
□ GitHub Actionsでworkflowを手動実行した
□ ワークフローが成功した（緑のチェックマーク）
□ Qdrantダッシュボードでデータを確認した
```

## 次のステップ

セットアップが完了したら：

1. **自動実行を待つ**: 毎日02:00 UTC（日本時間11:00）に自動実行されます
2. **ダッシュボード確認**: Qdrant Cloudのダッシュボードでデータの蓄積を確認
3. **ログ監視**: GitHub Actionsのログで異常がないか定期的にチェック

## サポート

問題が発生した場合：
1. [04.troubleshooting.md](./04.troubleshooting.md) を確認
2. GitHub Actionsのログを確認
3. GitHub Issuesで質問

## セキュリティに関する注意

- **APIキーやトークンを絶対にコミットしない**
- `.env`ファイルは`.gitignore`に含まれていることを確認
- GitHub Secretsのみを使用してクレデンシャルを管理
- 定期的にAPIキーをローテーション（推奨：3-6ヶ月ごと）
