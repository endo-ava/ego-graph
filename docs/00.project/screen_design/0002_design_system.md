# EgoGraph デザインシステム

## 1. デザインビジョン

EgoGraphは、個人のデジタルライフログを統合する「Personal Data Warehouse」です。
デザインは、以下の価値を体現します：

- **明晰性（Clarity）**: データを明確に、理解しやすく提示する
- **洗練性（Refinement）**: 極限まで削ぎ落とされた、タイポグラフィ重視の美しさ
- **未来感（Futurism）**: デジタルアーカイブとしての先進性と新鮮さ
- **信頼性（Reliability）**: 個人データを扱う製品としての安心感

## 2. デザインコンセプト「Digital Minimalism」

### コンセプト概要

**Digital Minimalism（デジタルミニマリズム）** は、情報の本質を際立たせる極限のシンプルさと、
サイバーシアンというシグネチャーカラーによる未来的アクセントを融合させたデザイン言語です。

### デザイン原則

1. **余白の力**: 余白は装飾ではなく、情報の呼吸を可能にする構造要素
2. **タイポグラフィファースト**: 文字の美しさとヒエラルキーが、UIの骨格を形成する
3. **シグネチャーカラー**: サイバーシアンを戦略的に配置し、視線を導く
4. **幾何学的整合性**: モダンで幾何学的なフォルムによる秩序と調和
5. **機能的アニメーション**: 動きは演出ではなく、理解を助ける機能

### 記憶に残る特徴

- **サイバーシアンのグロー効果**: アクセント要素に微細な発光エフェクトを適用
- **データグリッドパターン**: 背景に極薄のグリッドラインを配置し、データウェアハウス感を演出
- **モノスペースデータ表示**: 数値や日時をモノスペースフォントで表示し、精密性を強調

## 3. カラーシステム

Tailwind CSS v4のOKLCH色空間を採用し、知覚的に均一な色表現を実現します。

### 3.1 カラーパレット定義

| トークン名                  | 役割                 | 色調イメージ                                 |
| --------------------------- | -------------------- | -------------------------------------------- |
| `background` / `foreground` | ベースカラー         | 純白に近い白 / 深いブルーグレー (Dark: 反転) |
| `accent`                    | シグネチャーカラー   | **サイバーシアン** (鮮やかなシアン)          |
| `secondary`                 | 背景のバリエーション | 非常に明るいグレー / わずかに明るい背景      |
| `muted`                     | 控えめな要素         | 明るいグレー / 中間グレー                    |
| `border` / `input`          | 境界線               | 薄いグレー                                   |
| `destructive`               | 破壊的アクション     | レッド                                       |

### 3.2 カラー使用原則

1. **モノクロマティックベース**: 90%以上の面積をグレースケールで構成
2. **戦略的アクセント**: サイバーシアンは以下に限定使用
   - CTA（Call-to-Action）ボタン
   - 現在選択中の項目
   - 重要な通知・ハイライト
   - フォーカス状態のリング
3. **グロー効果**: アクセントカラーに微細な `box-shadow` でグロー演出（ダークモードで顕著に表示）

## 4. タイポグラフィシステム

### 4.1 フォントファミリー

| 役割             | フォント名         | 特徴                                                    |
| ---------------- | ------------------ | ------------------------------------------------------- |
| **プライマリ**   | **DM Sans**        | 幾何学的でモダンなサンセリフ。DisplayからBodyまで使用。 |
| **モノスペース** | **JetBrains Mono** | 日時・数値・コード表示用。技術的な精密性を強調。        |

### 4.2 タイポグラフィスケール

8pxベースのスケールを採用し、リズムを作ります。

- サイズ: `xs` (12px) ～ `5xl` (48px)
- ウェイト: `normal` (400) ～ `bold` (700)

## 5. レイアウトシステム

### 5.1 グリッドとスペーシング

- **8pxベース**: 全てのスペーシングは8の倍数を基本とします。
- **グリッド**: デスクトップ12カラム、モバイル4カラムを基本とします。

### 5.2 レイアウト原則

1. **余白重視**: コンテンツ間に十分な余白を確保し、圧迫感を排除
2. **最大幅制御**: テキストコンテンツは読みやすさのため `65ch` 程度に制限
3. **セーフエリア**: モバイルデバイスのセーフエリアを考慮

### 5.3 コンテナ

- **メインコンテナ**: 最大幅 `1200px`、中央寄せ、左右にパディング。
- **テキストコンテナ**: テキストの読みやすさを考慮し、最大幅 `65ch`。
- **セーフエリア対応**: モバイルデバイスのノッチやホームインジケーター領域を考慮したパディング。

## 6. コンポーネント設計原則

### 6.1 基本形状

- **ボーダーラディウス**: ミニマルデザインに合わせ、控えめな角丸（`sm` ～ `xl`）を採用。
- **シャドウ**: 最小限に留め、データグリッドパターンやグロー効果で奥行きを表現。

### 6.2 主要コンポーネント

#### ボタン (Button)

- **Primary**: サイバーシアン背景、ホバー時にグロー効果 + 上昇アニメーション
- **Secondary**: 背景色（secondary）、枠線あり
- **Ghost**: 背景透明、ホバー時に背景色出現

#### インプット (Input)

- 背景色は `background` を基本とし、`border` カラーの枠線を採用。
- フォーカス時に `accent` カラーのリングと `accent-glow` を表示。

#### カード (Card)

- `card` 背景色と `border` 境界線で構成。
- 必要に応じて控えめなシャドウを適用。

## 7. モーション・アニメーション

**「控えめ・機能的」** を哲学とし、演出ではなく理解を助けるために使用します。

### 7.1 アニメーション哲学

- **シンプルなトランジション**: フェード、スライドのみ
- **短いデュレーション**: 200-300ms
- **イージング**: `ease-out`（開始が速く、終了が滑らか）

### 7.2 トランジション設定

- **標準トランジション**: `150ms` (fast), `200ms` (base), `300ms` (slow) の `ease-out` を基本とします。

### 7.3 標準アニメーション

| 種類         | 用途               | 特徴                     |
| ------------ | ------------------ | ------------------------ |
| **Fade In**  | 画面遷移、要素出現 | 透明度のみの変化         |
| **Slide Up** | モーダル、通知     | 下から上への穏やかな移動 |
| **Glow**     | ホバー、フォーカス | 発光強度の変化           |

### 7.4 使用ガイドライン

| 状況                  | アニメーション                | デュレーション |
| --------------------- | ----------------------------- | -------------- |
| ページ/ビュー切り替え | フェードイン                  | 200ms          |
| モーダル登場          | フェードイン + スライドアップ | 250ms          |
| トースト通知          | スライドアップ                | 250ms          |
| ボタンホバー          | transform + box-shadow        | 150ms          |
| インプットフォーカス  | border-color + box-shadow     | 150ms          |

## 8. 実装リファレンス

実装時は以下のツール・ライブラリ設定を参照してください。

- **Tailwind CSS v4**: `@theme` ブロックによるデザイントークン定義
- **Google Fonts**: DM Sans, JetBrains Mono
- **shadcn/ui**: ベースコンポーネントとして利用し、スタイルをカスタマイズ

### 8.1 実装チェックリスト

- [ ] `src/index.css` にカラーパレット・フォント設定を追加
- [ ] Google Fontsから DM Sans と JetBrains Mono をインポート
- [ ] ボタンコンポーネントに `accent-glow` クラスを適用
- [ ] データグリッドパターンを背景に適用（オプション）
- [ ] アクセントカラーをCTAボタン・フォーカスリング・選択状態に限定使用
- [ ] アニメーションは `animate-fade-in` / `animate-slide-up` を使用
- [ ] モノスペースフォントを日時・数値表示に適用

## 10. まとめ

EgoGraphのデザインシステム「Digital Minimalism」は、以下を実現します：

- **極限のシンプルさ**: 情報の本質を際立たせる
- **サイバーシアンのシグネチャー**: 記憶に残る未来的アクセント
- **タイポグラフィファースト**: モダン幾何学的な美しさ
- **機能的アニメーション**: 理解を助ける控えめな動き

このデザインシステムに従うことで、EgoGraphは汎用的なAIデザインを超越した、
唯一無二のプロダクト体験を提供します。
